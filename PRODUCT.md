# Product

## Register

product

## Users

Consultants at client sites who need fast mental recall of how AI tools integrate, what they conflict with, and what architectural patterns they form. They are already competent — this is a reference tool, not a tutorial. They use it to think and confirm, not to be taught. Speed of recall matters more than depth of explanation.

## Product Purpose

Architectural Chromatics is an AI stack composition reference tool built around a color theory metaphor: tools are pigments, architectural roles are hues, stack combinations are harmonies. It lets a consultant quickly understand how a set of tools fit together, what tensions exist, and which canonical patterns apply. Success looks like a consultant arriving at an answer in under 60 seconds without needing to search outside the tool.

## Brand Personality

Playful, vivid, smart. The color metaphor is the product's intellectual engine, not a decoration. Playfulness comes from the chromatic richness and the metaphor itself — not from rounded corners, bubble-gum fills, or approachable-at-the-cost-of-serious aesthetics. Vibrancy comes from the hue system: every color is architectural. Intelligence shows through density, precision, and content quality — not through explainer copy.

References: Linear (craft floor — precision, trust, no wasted motion, interaction quality) and Observable (color-expressive data visualization done with restraint and intelligence).

## Anti-references

- No-code builder UI: over-rounded, pastel, bubble-gum. Approachable at the cost of serious.
- Dark mode for aesthetics: neon on black, cyberpunk vibes, dark because tools "look cool dark."
- AI product clichés: purple gradients, glowing orbs, neural-network backgrounds, ChatGPT-wrapper aesthetic.
- SaaS marketing landing: hero metrics, gradient text, "Book a Demo" energy — flashy and hollow.

## Design Principles

1. **Color earns meaning.** Every chromatic choice in the interface maps to the hue system. Indigo, amber, teal are semantic (accent, complexity, trust). No color is decorative. If it isn't architectural, it's gray.
2. **The metaphor is load-bearing.** Color theory isn't a theme — it's the reasoning system. Visual decisions must reinforce the metaphor, not sit alongside it as ornament.
3. **Consultant speed.** The interface serves recall, not discovery. An answer that takes three clicks has failed. Density is a feature when it saves time; clutter is not density.
4. **Vivid but grounded.** The hue palette is rich and expressive. The surrounding chrome is neutral. The tool's chromatic energy comes entirely from the data, not from the interface's decoration.
5. **Reference quality, not reference quantity.** The 11-tool, 7-hue, curated model is deliberate. The interface should feel equally curated: nothing in the UI should feel like it's there because it was easy to add.

## Accessibility & Inclusion

WCAG AA minimum. Hue meanings must never be conveyed by color alone — always paired with a label, shape, or positional signal. Reduced-motion preference should suppress non-state-change transitions. Color-blind users must be able to distinguish hue roles through secondary encoding (position on wheel, label text, icon).
